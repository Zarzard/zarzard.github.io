CNN(Convolutional Neural Network)

Main Task

训练一个 classifier，进行模式识别（判别图片上的人是男人 or 女人等等）

（这个 classifier 实际上是一个多层 NN，即 CNN。输出只有 -1（正例） 或 1（负例），本质上每一个neuron 都代表着一个二分类器，超过阈值被激活则说明在这个 pattern 上符合正例的某个 pattern）

Basic Process

raw image $\rightarrow$ Convolution $\rightarrow$ Max_pooling $\rightarrow$ Convolution $\rightarrow$ Max_pooling $\rightarrow\cdots$

$\rightarrow$ Flatten (变成普通的 fully connected NN)

$\rightarrow$ Fully connected feed-forward network

Motivation

Naive image classifier：把每一个像素都当成一个 feature。

问题：待学习 param 的数量过多，即权重矩阵维数过大（实际上，光是连接 input layer 与 第一个 hidden layer 的轴突数量就是图像矩阵维数的好几倍了）

Observation（Why CNN works?)

  • 例子1：对于一个图像分类问题：判断是男人还是女人。一些神经元负责判断头发长短，另一些神经元负责判断胸部特征（以上这两个都称为”pattern”），那么这些神经元并不需要 input layer 的全部像素，而只需要局部的像素来帮助 classify；
  • 例子2：在上一个例子的基础上，再来一张图片（长发女人的）判断男女，如果上一张图片也是女人而且也是长头发，但是在图片中的相对位置不一样，那么我们希望能用到上一张的判断结果;；
  • 例子3：间隔抽去图像像素矩阵的各行，仍然能判断出图像的物体是什么。

Idea

首先总结上述 Observation：

  1. Some patterns are much smaller than the whole image;
  2. A pattern appear in several images, but in different locations of each image;
  3. Subsampling the pixels will not change the object in the images.

对于 1, 2 两点，我们用 Convolution 来做；对于 3，用 Max_pooling.

（两个操作都是在减少 fully connected 的 naive image classifier 中的待学习参数）

Convolution

HOW

对一个 6*6 的 pixel 矩阵，用一个 3*3 （3 是 hyper param）的 filter（其上各个元素是模型参数，即待学习参数）在 pixel 矩阵上移动(stride)，依次与重叠的部分做欧式内积（把矩阵展成向量做内积），得到一个 4*4 的矩阵

WHY

以黑白图像（二维 tensor）为例，每一个 filter 都是需要学习出来的 neuron，代表了一个大小为 33（针对这个例子）的 pattern，并可以作为一个 classifier 用来 *侦测 这个 pattern。

怎么侦测呢？pixel 矩阵中，0 代表 黑，1 代表 白；filter 中的 1 代表正例，-1 代表负例。那么通过 filter 与重叠部分的点乘后累加，得到的值越大（最理想的是 1 与 1 完全重叠，-1 与 0 完全重叠），说明这个部分的 pixel 矩阵越符合这个 pattern 。

把整个 pixel 矩阵拉成一个 vec，看成 input layer；把 filter 处理后的 6*6 矩阵看成第一层 hidden layer，把 filter 各个元素看成 input layer 与 hidden layer 之间的 weights（见下图），则实际上 Convolution 这个操作就是构造一个 not fully connected 的神经网络 .

注：

  • 用维数比 pixel 矩阵小得多的 filter，体现的是第一个 Obser

  • 用这个 filter 在 pixel 矩阵上 stride，侦测这张图上不同地方有没有这个 pattern，体现的是第二个 Obser

  • 当然，实际做的时候，每一层的 neuron 都不是只由一个 filter 得到的，而是多个 filter 得到的叠加起来，称为 “feature map”。这样得到的 “feature map”可看成一个根据某个 pattern 得到的压缩图片，只是这个 pixel 矩阵是有“深度”的，这个深度就是 filter 的数量（如下图深度为 2 ，说明用了两个 filter）

Max_pooling

HOW

对上面每一个 filter 得到的 4*4 矩阵，取一个 2*2（2是 hyper param）的小方格，在上面移动（当然这次与 convolution 不同，每次重叠部分不相交），依次取重叠部分中最大的那个元素，作为小方格中四个元素的代表，得到一个 2*2 的矩阵

WHY

这是一种 subsampling。每个小框内只使用最“像”该 pattern 的元素作为训练样本（把 input layer $\rightarrow$ first hidden layer 的过程看成 subsampling 预处理）

fully connected VS convolution

  • convolution 减少了神经元的连接数(即 fully connected 的边数)，减至 filter 矩阵(mn维) 的 size（即mn)。每个 neuron 只连接与它做欧式内积的，图像上的一个 m\n 维 pixel 矩阵；
  • Weight(filter) Sharing：每一层各个神经元的 weight 的数量以及 weight 本身的数值（即 filter）是相同的，不同的只有连接的神经元

Intuitions

每一个 filter 代表了一个局部的 pattern，所以让这个 filter 最 activate 的图片就是这种不断重复这种 pattern（其实就是某种纹路）图片。不同的 filter 负责 detect 不同角度的纹路。

与 Convolution 不同，由于 Flatten 后 Hidden layer 里的每一个 neuron 与输入层是 fully connected 的，这意味着 Hidden layer 里的每一个 neuron 代表的都是与原始图片一样大的 pattern。即：使每一个 neuron 最 activate 的图片是其对应的一个整体的 pattern。

围棋

为什么可以用？因为围棋自然可以看成 19*19 的 image，并且关键是：也具备上面 Observation 1, 2 的特点。不过由于没有第三个特点，所以不能用 subsampling。

声音处理（判别）

看频谱（纵轴为频率，横轴为时间）

只需要在纵轴方向 stride filter 即可，因为 tempo 方面后面会有 LTSM 等等来做，而频率方面可以有类似于图像的 “pattern”，如：男生女生说同一句话可以只是在音高上不一样，但是其只是差一个八度那样子，频率的“形状”是一样的。

文本处理（判别感情正负）

为什么 text 也可以处理成一个矩阵呢？把每个 word embedding 合起来就变成了一个矩阵！

那么这里的 filter 长啥样？要怎么 stride 它呢？

答案是：把 filter 取成高度为 embedding 的维数的矩阵，只在句子方向移动 filter 。

为什么不在 embedding 的各 dimension 之间找 pattern 呢？

因为在 word2vec 里面， vec 的各个维度其实是 independent 的，相互之间没有依赖性（对比图像中的车子的像素之间显然是有 dependence 的），也就没有 pattern。而我们用CNN是为了找到 pattern 以判断整个 text 的感情正负，所以不用这个方向的移动。

感受野

定义：在CNN 中，每一层的输出由输入的某些部分生成（比如左上角的一个，由输入的左上4个和卷积核做互相关运算得到），这些部分称为“x 在 （输入）上的感受野”。

Padding(填充)

在原图的周围添加数行0（具体多少行看卷积核，比卷积核长宽各少一就对了），使得输入和输出具有相同的高和宽

Channels

若卷积核（filter）使用和输入相同的channels数，即形状为 c*h*w，那么输出就是一个channel（各层卷积后对应相加）

如果想要得到多个channel的输出，那么就需要对 filter 多加一个维度，c1*c*h*w，每个 c*h*w 得到一个输出 channel，则 c1 个 c*h*w 得到 c1 个输出 channel 。
